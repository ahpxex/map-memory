# map-memory PRD v0

## 1. Product Summary

`map-memory` 是一个纯前端、纯离线运行的交互式地图记忆训练网页。

它的目标不是做地图浏览器、地理百科站或内容门户，而是帮助用户通过主动回忆，
真正记住：

- 世界上每一个国家
- 中国每一个地级行政区

整个产品采用单屏结构：

- 地图主画布占据几乎全部可视区域
- 底部只有一个浮动工具栏
- 点击地图区域后，以轻量 popup 呈现当前区域的上下文信息或训练反馈

## 2. Product Goals

### 2.1 Primary Goals

- 让用户能够在地图空间位置和名称之间建立稳定记忆
- 用统一的交互框架同时承载 `世界国家` 与 `中国地级行政区` 两套数据
- 保持界面极简，避免 sidebar、header、内容列表分散注意力
- 在完全离线的前提下，保留用户训练数据的本地持久化，以及导入导出能力

### 2.2 Success Criteria

- 用户能够在不打开 label 的情况下，逐渐提升对区域位置和名称的回忆能力
- 世界模式与中国模式的交互方式尽可能一致，不形成两套产品体验
- 工具栏可以覆盖主要的模式调节需求，而不需要额外页面
- 用户可以导出训练数据，在另一台机器或浏览器导入后继续使用

## 3. Non-Goals

以下内容不属于首版目标：

- 不做后端
- 不做账号系统或云同步
- 不做排行榜、社交功能、多用户协作
- 不做内容型百科页或长篇资料页
- 不做大量装饰性 UI 面板
- 不默认扩展到“所有地图知识”，例如山脉、河流、省界、首都、旗帜等

## 4. Target User

首要目标用户就是项目发起人本人：

- 想记住世界上每一个国家
- 想记住中国每一个地级行政区
- 更关心“记住”而不是“看过”

次级用户是：

- 对地图、行政区划、地理空间记忆有兴趣的人
- 不喜欢表格式死记硬背，希望通过可交互训练建立空间认知的人

## 5. Core Product Principles

### 5.1 Map First

地图是唯一主舞台。除了底部工具栏和局部 popup，不引入长期占据视野的其他固定 UI。

### 5.2 Recall Over Browsing

产品核心是主动回忆训练，不是地图展示。所有模式设计都要优先服务“回忆”。

### 5.3 One Interaction Framework, Two Datasets

`世界国家` 和 `中国地级行政区` 只是两套不同数据，不应该演化成两套完全不同的产品。

### 5.4 Offline By Default

产品应在无网络情况下继续工作。运行时不依赖远程 API。训练记录、用户偏好、错题进度保存在本地。

### 5.5 User Owns the Data

既然没有后端和账号系统，就必须提供数据导出和导入，让用户对自己的训练记录拥有可迁移性。

## 6. App Structure

整个应用由三层组成：

### 6.1 Layer A: Map Canvas

地图主画布占满主要视图区域，支持：

- 拖拽平移
- 缩放
- hover 高亮
- click 选中区域

### 6.2 Layer B: Bottom Floating Toolbar

底部居中悬浮，不占据全宽页面，不做成传统 footer。

它负责修改全局状态，例如：

- 切换地图模式：`世界国家` / `中国地级行政区`
- 切换交互模式：`探索` / `训练`
- 切换训练子模式：`看图猜名` / `看名点图` / `错题回放`
- 是否显示 label
- 语言设置
- popup 内容密度
- 重置视图
- 打开导入 / 导出

### 6.3 Layer C: Contextual Popup

点击地图区域后出现，锚定在区域附近或视窗安全位置。

popup 是局部反馈容器，不是侧边详情页。它根据当前模式显示不同内容。

## 7. Primary Modes

## 7.1 Dataset Mode

这是第一层模式：

- `世界国家`
- `中国地级行政区`

两者共用同一套交互框架，但数据口径不同。

### 世界国家模式

- 基本单位：主权国家 / 训练口径约定下的国家对象
- 地图层级简单，主要关注国家名称、位置、邻接关系

### 中国地级行政区模式

- 基本单位：地级市、自治州、地区、盟等同级行政单位
- 用户默认看到的是地级层级，而不是省级层级
- 需要明确上级省级行政区归属

## 7.2 Interaction Mode

这是第二层模式：

- `探索模式`
- `训练模式`

### 探索模式

目标：熟悉地图、浏览 label、查看基础概况。

行为：

- 点击区域后显示基础信息 popup
- 可打开 label 辅助学习
- 更像“可交互地图学习器”

### 训练模式

目标：进入主动回忆与反馈循环。

行为：

- popup 更偏向答案反馈与训练控制
- label 通常默认关闭
- 用户以正确率、错题、熟练度为主线推进

## 7.3 Training Submodes

在 `训练模式` 下，首版至少支持以下子模式：

### 看图猜名

- 地图高亮一个区域
- 用户需要说出或选择它的名称
- 回答后显示对错与正确答案

### 看名点图

- 系统给出一个地区名称
- 用户在地图上点击正确区域
- 点击后反馈对错

### 错题回放

- 优先出用户经常出错、或近期答错的区域
- 支持针对错题集中强化

## 8. Toolbar Design

工具栏是产品的唯一全局控制中心。

### 8.1 Toolbar Requirements

- 悬浮在底部中央
- 自适应桌面与移动端
- 折叠时保持极简
- 展开时可分组展示更多控制项

### 8.2 Toolbar Information Groups

首版建议按以下分组组织：

- 地图范围
  - 世界
  - 中国

- 交互模式
  - 探索
  - 训练

- 训练子模式
  - 看图猜名
  - 看名点图
  - 错题回放

- 显示辅助
  - label on/off
  - 边界强调
  - 颜色强度

- 语言
  - 中文
  - 英文
  - 中英混合

- 数据
  - 导出训练数据
  - 导入训练数据
  - 清空本地训练数据

## 9. Popup Rules

popup 的内容由当前模式决定，而不是简单绑定某一个 toggle。

## 9.1 Explore Popup

在 `探索模式` 下，popup 主要显示：

- 当前名称
- 其他语言名称 / 别名
- 上级归属
- 邻接信息
- 基础概况

基础概况首版可以非常轻，只要支持学习，不必做成百科卡片。

例如：

- 世界国家：所属洲、首都、邻国数量
- 中国地级行政区：所属省级单位、行政类型、邻接地区数量

## 9.2 Training Popup

在 `训练模式` 下，popup 主要显示：

- 当前题目信息
- 用户答案与系统判定
- 正确答案
- 下一题入口
- 是否加入错题池 / 是否已记录

训练 popup 的信息密度应明显低于探索 popup，避免用户在训练中被信息干扰。

## 9.3 Label Interaction With Popup

`是否显示 label` 只是显示辅助开关，不应直接决定 popup 的信息结构。

但它可以影响 popup 的默认内容策略：

- 当 label 开启时，探索 popup 可以展示更多基础概况
- 当 label 关闭时，训练 popup 应更偏简洁反馈

## 10. Map Interaction Details

地图交互必须自然，不应让用户感觉像在操作复杂 GIS 工具。

### 10.1 Required Interactions

- 鼠标滚轮 / 双指缩放
- 拖拽平移
- hover 高亮
- click 选中
- 点击空白处取消选中

### 10.2 Visual States

每个区域至少有以下状态：

- default
- hover
- selected
- correct
- wrong
- muted

这些状态在世界地图和中国地图里都应保持一致语义。

## 11. Offline Data Model

本产品纯离线运行，因此需要明确区分两类数据：

### 11.1 Built-in Static Data

由产品内置，随应用打包：

- 地图几何数据
- 区域基础元数据
- 多语言 label
- 区域之间的邻接或父级关系

### 11.2 User Local Data

保存在本地浏览器，至少包括：

- 用户偏好设置
- 当前地图模式和工具栏配置
- 历史答题记录
- 错题池
- 熟练度 / 最近练习时间

## 12. Import / Export

这是首版必须支持的能力，而不是后补功能。

## 12.1 Why It Matters

因为产品没有后端和账号系统，如果没有导入导出，用户训练数据将被锁死在当前浏览器环境中。

## 12.2 Export Scope

用户应可以导出自己的本地训练数据为单个文件。

首版建议导出内容包括：

- app version
- dataset progress
- user settings
- error history
- mastery / review state
- export timestamp

### 12.3 Export Format

首版优先采用：

- `JSON`

理由：

- 可读
- 易调试
- 易兼容未来版本迁移
- 不依赖后端

### 12.4 Import Rules

导入时需要：

- 校验文件是否合法
- 校验版本是否兼容
- 明确告知用户导入会覆盖还是合并本地数据

首版建议默认策略：

- 支持 `replace import`
- 预留未来 `merge import` 的空间

### 12.5 Safety Rules

导入前应给出确认：

- 导入后是否覆盖当前本地训练记录
- 如果文件版本过旧或过新，给出提示
- 如果缺字段或损坏，拒绝导入并说明原因

## 13. Technical Constraints

### 13.1 Architecture

- 纯前端
- 不依赖后端 API
- 尽量避免运行时网络请求
- 所有核心训练流程在浏览器本地完成

### 13.2 Storage

首版本地存储建议：

- `localStorage` 或 `IndexedDB`

建议判断：

- 如果数据结构只包含轻量设置和有限训练记录，`localStorage` 足够
- 如果未来要存更多训练历史、错题明细和更复杂的 review 状态，优先 `IndexedDB`

当前建议：

- 从 `IndexedDB` 起步，避免后续迁移成本

### 13.3 Data Abstraction

世界国家和中国地级行政区都应抽象成统一的 `RegionFeature` 结构，至少包括：

- `id`
- `dataset`
- `geometry`
- `labels`
- `aliases`
- `parentId`
- `neighbors`
- `metadata`

用户训练状态应与区域定义解耦，避免把训练数据耦合进地图静态数据本身。

## 14. This Week Focus

这是一个 weekly project。

因此这里不采用 `MVP`、`sprint goal` 这种创业项目或团队节奏里的压缩叙事，
而是更实际地回答：

- 这周先把什么做出来最值
- 这周哪些东西没必要现在锁死
- 到周末时什么样的完成度算是一个好的 week 13 版本

### 14.1 This Week Should Produce

本周最值得先做出的闭环是：

- 两套地图数据都能加载
- 底部浮动工具栏可切世界 / 中国
- 地图可缩放、拖拽、点击
- popup 可根据探索 / 训练模式切换内容
- 支持 label 开关
- 支持中英至少两种语言显示
- 支持 `看名点图`
- 支持本地训练记录持久化
- 支持导出 / 导入训练数据

### 14.2 This Week Does Not Need To Over-Decide

本周不需要急着把这些事情做重：

- 极复杂的 spaced repetition 算法
- 很重的百科信息
- 动画炫技
- 海量统计面板

### 14.3 Good Week 13 Outcome

如果到本周后段，产品已经达到下面这个状态，就算是很好的 week 13 版本：

- 可以在世界 / 中国两张地图之间切换
- 只有地图、底部工具栏和 popup 这三层核心结构
- 用户已经能进行至少一种真正的交互式记忆训练
- 本地数据能保存，并且能导出、导入

## 15. Later Expansion

等这周的核心体验站稳之后，再考虑：

- 更细的熟练度模型
- 更丰富的错题回放策略
- 自定义训练集
- 热键驱动训练
- 省级 / 州级 / 首都 / 旗帜等扩展训练层
- 更完整的统计视图

## 16. Open Questions

以下问题仍需在后续迭代中锁定：

- 世界模式的国家口径具体如何定义
- 中国地级行政区数据源选哪套口径
- popup 的默认锚定和移动端表现如何设计
- 首版训练输入是自由输入、模糊匹配，还是先用点选 / 选择
- 工具栏在移动端是横向滑动、分页，还是抽屉展开

## 17. Recommended Next Step

下一步不该直接开始堆功能，而应该先把下面三件事收束清楚：

- 页面信息架构和状态图
- 数据结构与导入导出 schema
- 本周最先实现的训练闭环到底选哪一个

当这三件事清楚之后，再进入 UI 具体实现和地图数据接入。
