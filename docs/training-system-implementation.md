# 地图学习系统实施文档

## 1. 文档目的

这份文档定义 `map-memory` 当前版本应实现的完整学习系统。

目标不是继续停留在“给一个地名，让用户去地图上点出来”的单一刷题模式，而是把产品升级为一个真正的地图记忆训练系统：

- 世界地图：训练国家位置、国旗、首都、大洲/次区域、邻国关系。
- 中国地图：训练地级市/州位置、所属省份、同省专项、省会、省级邻接关系。
- 所有训练都支持范围筛选，不再只做全图随机练习。
- 所有训练都接入统一的记忆调度、错题复练、薄弱项复练和考试模式。

本版本不采用“先做一点、后续再补”的拆分策略。本文内的内容均属于当前版本范围。

## 2. 产品定义

### 2.1 产品定位

`map-memory` 是一个主动回忆型地图学习产品，不是地图浏览器，也不是百科站。

它训练的核心能力有四类：

- 识别：看到名称、国旗、首都、区域信息，能定位到地图对象。
- 回忆：看到地图对象，能回忆出它的名字、旗帜、首都、所属关系。
- 关系：知道国家之间、城市与省份之间的结构关系。
- 干扰辨析：能区分相似国家、相似城市、相邻国家、相邻省份等高混淆项。

### 2.2 当前实现的局限

当前代码中，训练模式本质上只有一个：

- `locate-from-name`

也就是：

- 系统给一个地区名称。
- 用户在地图上点击对应地区。
- 系统只判断点对或点错。

这套机制可以保留，但它只能训练“名字 -> 位置”的单向识别，不足以构成完整的记忆闭环。

## 4. 产品总结构

训练系统拆成四层：

### 4.1 Dataset

- `world`
- `china`

### 4.2 Mode

Mode 决定题怎么出、怎么答、怎么判。

### 4.3 Scope

Scope 决定从哪一批地区中出题。

世界地图 Scope：

- 全部国家
- 按大洲
- 微型国家专项
- 易混国家专项
- 错题本
- 薄弱项

中国地图 Scope：

- 全国
- 按省份
- 同省专项
- 易混城市专项
- 错题本
- 薄弱项

### 4.4 Skill

Skill 是掌握度记录的最小单位。

同一个地区在不同技能下必须分别记录，不允许把“位置会了”误当成“首都会了”或“国旗会了”。

## 5. 世界地图题型

### 5.1 看名找图

- 题面：给国家名
- 作答：在地图上点击国家
- 判题：点击的国家是否与目标国家一致
- 目标技能：`location`

### 5.2 看图选名

- 题面：高亮一个国家
- 作答：从四个国家名里选对
- 判题：选项是否正确
- 目标技能：`shape_name`

### 5.3 国旗找国家

- 题面：给国旗
- 作答：在地图上点击对应国家
- 判题：点击是否正确
- 目标技能：`flag`

### 5.4 国家找国旗

- 题面：给国家名
- 作答：从四面国旗中选对
- 判题：选项是否正确
- 目标技能：`flag`

### 5.5 首都找国家

- 题面：给首都名
- 作答：在地图上点击对应国家
- 判题：点击是否正确
- 目标技能：`capital`

### 5.6 国家找首都

- 题面：给国家名
- 作答：从四个首都里选对
- 判题：选项是否正确
- 目标技能：`capital`

### 5.7 国家找大洲

- 题面：给国家名
- 作答：从若干大洲选项中选对
- 判题：选项是否正确
- 目标技能：`continent`

### 5.8 国家找次区域

- 题面：给国家名
- 作答：从次区域选项中选对
- 判题：选项是否正确
- 目标技能：`subregion`

### 5.9 邻国判断

- 题面：给两个国家
- 作答：判断是否接壤
- 判题：是否正确
- 目标技能：`neighbors`

### 5.10 邻国连击

- 题面：给一个国家
- 作答：连续点击它的所有邻国
- 判题：每次点击都要判断是否属于目标国家的邻国集合
- 目标技能：`neighbors`

### 5.11 易混国家专项

- 题面：按指定模式生成
- 候选池：只从混淆组中取
- 目标技能：随题型变化

### 5.12 微型国家专项

- 题面：按指定模式生成
- 候选池：只从微型国家集合中取
- 目标技能：随题型变化

## 6. 中国地图题型

### 6.1 看名找图

- 题面：给地级市/州名
- 作答：在地图上点击地区
- 判题：点击是否正确
- 目标技能：`location`

### 6.2 看图选名

- 题面：高亮一个地级市/州
- 作答：从四个地级市/州名称中选对
- 判题：选项是否正确
- 目标技能：`shape_name`

### 6.3 城市找所属省份

- 题面：给地级市/州名
- 作答：从省份列表中选对所属省
- 判题：选项是否正确
- 目标技能：`province_affiliation`

### 6.4 省份找下属城市

- 题面：给一个省
- 作答：在地图上连续点出属于该省的城市
- 判题：点击是否属于当前省份
- 目标技能：`province_children`

### 6.5 同省专项

- 题面：用户先选一个省，再在该省范围内进行多个题型训练
- 作答：取决于子题型
- 判题：取决于子题型
- 目标技能：覆盖 `location`、`shape_name`、`province_children`

### 6.6 省会训练

- 题面一：给省份，选省会
- 题面二：给省会，选对应省份
- 判题：选项是否正确
- 目标技能：`provincial_capital`

### 6.7 邻省判断

- 题面：给两个省
- 作答：判断是否接壤
- 判题：是否正确
- 目标技能：`neighbors`

### 6.8 易混城市专项

- 题面：按指定模式生成
- 候选池：只从混淆组中取
- 目标技能：随题型变化

## 7. Scope 设计

### 7.1 世界地图 Scope

- `all`
- `continent`
- `microstates`
- `confusion-set`
- `wrong-only`
- `weak-only`

当 `scope = continent` 时，必须记录 `scopeValue`：

- `asia`
- `europe`
- `africa`
- `north-america`
- `south-america`
- `oceania`

### 7.2 中国地图 Scope

- `all`
- `province`
- `same-province`
- `confusion-set`
- `wrong-only`
- `weak-only`

当 `scope = province` 或 `same-province` 时，必须记录 `scopeValue`：

- 对应省份 id

### 7.3 Scope 规则

- 世界地图所有题型都必须支持按大洲筛选。
- 中国地图所有题型都必须支持按省份筛选。
- `同省专项` 必须要求先选省。
- 切换 Scope 后，题目候选池立即重算。
- Scope 状态必须持久化。

## 10. 题目调度规则

### 10.1 调度步骤

每次出题按以下顺序执行：

1. 根据 `dataset` 选数据集
2. 根据 `scope` 过滤候选池
3. 根据 `mode` 和 `skill` 过滤可用数据
4. 根据掌握度排序
5. 从最高优先级候选池中带随机性抽题

### 10.2 优先级原则

- 从未练过的优先
- 最近做错过的优先
- 长期未复习的优先
- 掌握度低的优先
- 连续答对很多次的降权

### 10.3 选择题干扰项规则

世界地图干扰项优先从这些来源里取：

- 同大洲
- 同次区域
- 邻国
- 同混淆组

中国地图干扰项优先从这些来源里取：

- 同省
- 邻省
- 同混淆组
- 名称结构相似项

规则：

- 不能使用纯随机弱干扰项
- 选项难度要可控
- 每种题型都要定义干扰项生成策略

## 11. 通用学习机制

### 11.1 错题本专练

- 只从某技能下答错过的对象中出题
- 必须支持世界和中国
- 必须支持与 Scope 叠加

### 11.2 薄弱项专练

- 只从 `masteryScore` 低于阈值的对象中出题
- 阈值需要统一定义

### 11.3 混合复习

候选题按比例混合：

- 新题
- 错题
- 薄弱题
- 已掌握题

默认比例可配置，但要有项目内固定默认值。

### 11.4 考试模式

- 固定题量
- 不立即显示正误
- 结束后统一结算

### 11.5 限时挑战

- 固定时间
- 在时间内尽量答题
- 以速度和正确率综合计分

### 11.6 连对挑战

- 要求连续答对若干题
- 一旦答错，连续计数中断

## 13. 结果反馈规则

### 13.1 即时反馈

每种题型都必须输出具体反馈，不允许只有“对/错”。

### 13.2 错误反馈内容

至少要包含：

- 用户给出的答案
- 正确答案
- 当前题型下正确的知识关系

示例：

- 你点的是 `A`，正确答案是 `B`
- `安庆市` 属于 `安徽省`，不是 `江苏省`
- `布达佩斯` 是 `匈牙利` 的首都
- `A` 与 `B` 不接壤

### 13.3 题后操作

题后至少支持：

- 下一题
- 重试本题
- 加入标记
- 切到错题本

这里的“顺序”只是工程依赖顺序，不代表版本拆分；本文所有内容都属于当前版本交付范围。

## 18. 最终定义

本版本交付完成后，`map-memory` 不再是：

- 一个只能做“给地名找位置”的单向地图点击器

而应该是：

- 一个支持世界地图与中国地图、多题型、多范围、多技能维度、带复习机制和挑战机制的完整地图学习系统
